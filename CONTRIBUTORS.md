# Contributors

Thank you for contributing to **SLTM**. This document describes how we recognize contributors and how to add yourself to the list.

## How to contribute

There are many ways to contribute:

- Reporting bugs (with clear reproduction steps)
- Proposing features or improvements
- Improving documentation
- Submitting pull requests (fixes, features, refactors, tests)
- Helping others in issues/discussions and reviewing PRs

Before you start, please check:

- `README.md` for setup/run instructions
- `SECURITY.md` for reporting vulnerabilities responsibly
- `.github/ISSUE_TEMPLATE/` and `pull_request_template.md` for expected issue/PR details

## Recognition policy

We recognize contributors in this repository based on meaningful contributions, including (but not limited to):

- Merged pull requests
- Verified bug reports that lead to a fix
- Significant documentation improvements
- Ongoing maintenance/review work

We keep this list lightweight and manually curated. If you contributed and are missing, please add yourself following the instructions below.

## Add yourself to the list

1. Fork the repo and create a branch.
2. Edit this file: `CONTRIBUTORS.md`
3. Add your entry to the list below:
   - Keep entries **alphabetical by name**.
   - Use the format shown.
4. Open a pull request.

### Entry format

- **Name** — Role/Contribution area — GitHub: `@handle` — (optional) Link: https://...

Example:

- **Ada Lovelace** — Backend/API — GitHub: `@ada` — https://example.com
